# Contributing to GuideRAG 🤝

感谢你愿意为 GuideRAG 贡献代码！我们欢迎各种形式的贡献：从修复错别字、完善文档，到增加核心功能和修复 Bug。

## 如何开始？

1. **Fork** 本仓库。
2. **Clone** 你 Fork 出来的仓库到本地。
3. **Create a branch** (例如: `feat/amazing-feature` 或 `fix/bug-id`)。
4. **Build & Test** 确保你的改动是正确且通过编译的。
5. **Commit** 你的修改。
6. **Push** 分支到 GitHub。
7. **Create a Pull Request**。

## 代码规范
- **后端**: 请遵循 [Google Java Style Guide](https://google.github.io/styleguide/javaguide.html)。
- **前端**: 使用 Vue 3 `<script setup>` 模式，遵循 ESLint 与 Prettier 配置。
- **注释**: 新功能的代码请使用清晰的 **中文注释**（本项目推崇中文原生架构思想）。

## 提交规范 (Commit Messages)
请使用以下格式：
- `feat`: 新增功能
- `fix`: 修复 Bug
- `docs`: 文档更新
- `style`: 代码格式化 (不影响逻辑的变动)
- `refactor`: 重构
- `test`: 增加/修改测试代码
