## 1. 项目概述

**项目名称**：RAG 智库助手  
**前端技术栈**：Vue 3 + TypeScript + Vite + Pinia  
**后端技术栈**：Spring Boot + Java  
**对接目标**：将前端从使用模拟数据切换到调用真实后端 API

**主要功能模块**：
- 用户认证（登录/注册/退出）
- 会话管理（创建/查询/删除）
- 消息对话（发送/接收/流式响应）
- 知识库管理（CRUD）
- 文档管理（上传/解析/删除）

## 3. 核心差异对比

### 3.1 响应格式差异

| 项目 | 模拟数据 | 后端 API |
|-----|---------|---------|
| 成功码类型 | `number` (200) | `string` ("00000") |
| 成功判断 | `code === 200` | `code === "00000"` |
| 错误码 | `400, 401, 500` | `"A0001"`, `"B0001"` 等 |

### 3.2 字段名称映射

#### 通用字段

| 前端字段 | 后端字段 | 说明 |
|---------|---------|-----|
| `id` | `sessionId/kbId/docId` | 各资源有专属 ID 字段 |
| `createdAt` | `gmtCreate` | 创建时间 |
| `updatedAt` | `gmtModified` | 修改时间 |

#### 认证相关

| 前端字段 | 后端字段 | 说明 |
|---------|---------|-----|
| `username` (登录请求) | `account` | 账号 |
| `token` | `accessToken` | 访问令牌 |
| - | `refreshToken` | 刷新令牌（新增） |
| `user.id` | `user.userId` | 用户 ID |

#### 会话相关

| 前端字段 | 后端字段 | 说明 |
|---------|---------|-----|
| `session.id` | `session.sessionId` | 会话 ID |
| `message.id` | `message.messageId` | 消息 ID |

#### 知识库相关

| 前端字段 | 后端字段 | 说明 |
|---------|---------|-----|
| `kb.id` | `kb.kbId` | 知识库 ID |
| `kb.documentCount` | `kb.docCount` | 文档数量 |
| `doc.id` | `doc.docId` | 文档 ID |
| `doc.status` | `doc.parseStatus` | 解析状态 |

### 3.3 分页响应差异

**模拟数据**：
```typescript
{
  code: 200,
  data: [item1, item2, item3]  // 直接返回数组
}
```

**后端 API**：
```typescript
{
  code: "00000",
  data: {
    items: [item1, item2, item3],  // 数组在 items 字段中
    page: 1,
    size: 20,
    total: 100,
    totalPages: 5
  }
}
```

### 3.4 SSE 事件格式

**后端 SSE 格式**：
```
event: start
data: {"messageId":"m_123","sessionId":"s_456","timestamp":"..."}

event: delta
data: {"delta":"文本片段"}

event: citation
data: {"citations":[{"docId":"d_1","chunkId":"c_1","score":0.92}]}

event: done
data: {"usage":{"totalTokens":150},"finishReason":"stop"}

event: error
data: {"code":"E001","message":"错误信息"}
```

**解析关键**：
- 事件类型在 `event:` 行
- 数据在 `data:` 行，为 JSON 字符串
- 需要分别处理每种事件类型

## 4. 遇到的问题与解决方案

### 4.1 响应码类型不匹配

**问题**：
```typescript
// 编译错误：This comparison appears to be unintentional because 
// the types 'string' and 'number' have no overlap
if (response.code === 200) { }
```

**原因**：后端返回的 `code` 是字符串 `"00000"`，而前端判断使用数字 `200`

**解决**：
```typescript
// 方案 1：直接修改判断条件
if (response.code === '00000') { }

// 方案 2：统一在拦截器中处理
http.interceptors.response.use((response) => {
  if (response.data.code !== '00000') {
    throw new Error(response.data.message)
  }
  return response.data  // 直接返回 data，不需要业务代码判断
})
```

### 4.2 字段名称不一致

**问题**：前端使用 `id`，后端返回 `sessionId/kbId/docId`

**解决方案 1：数据映射**（推荐）
```typescript
const sessions = response.data.items.map(s => ({
  id: s.sessionId,        // 映射为前端期望的字段名
  title: s.title,
  updatedAt: s.gmtModified
}))
```

**解决方案 2：修改前端类型**（不推荐，改动太大）
```typescript
// 不推荐：需要修改所有使用该类型的地方
interface Session {
  sessionId: string  // 原为 id
  // ...
}
```

**建议**：在 API 层进行映射，保持业务层字段名不变

### 4.3 分页数据结构变化

**问题**：
```typescript
// 原代码期望直接获取数组
sessionStore.setSessions(response.data)  // 错误！data 是对象而非数组

// 编译错误：Property 'map' does not exist on type 'SessionListData'
```

### 4.4 SSE 流式响应解析

**问题**：后端 SSE 使用 `event:/data:` 格式，需要正确解析

**解决**：
```typescript
export function createChatStream(
  sessionId: string,
  query: string,
  kbId: string | undefined,
  callbacks: SSECallbacks
): () => void {
  const eventSource = new EventSource(url)
  
  // 监听各种事件类型
  eventSource.addEventListener('start', (e) => {
    const data = JSON.parse(e.data)
    callbacks.onStart?.(data)
  })
  
  eventSource.addEventListener('delta', (e) => {
    const data = JSON.parse(e.data)
    callbacks.onDelta?.(data)
  })
  
  eventSource.addEventListener('citation', (e) => {
    const data = JSON.parse(e.data)
    callbacks.onCitation?.(data)
  })
  
  eventSource.addEventListener('done', (e) => {
    const data = JSON.parse(e.data)
    callbacks.onDone?.(data)
    eventSource.close()
  })
  
  eventSource.addEventListener('error', (e) => {
    const data = JSON.parse(e.data)
    callbacks.onError?.(data)
    eventSource.close()
  })
  
  // 返回取消函数
  return () => eventSource.close()
}
```

## 5. 关键代码示例

### 5.1 统一响应处理

```typescript
// frontend/src/api/client.ts
import axios from 'axios'

export interface BackendResponse<T = any> {
  code: string
  message: string
  data: T
}

const http = axios.create({
  baseURL: '/api/rag/v1',
  timeout: 30000
})

// 请求拦截器：添加 Token
http.interceptors.request.use((config) => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

// 响应拦截器：统一处理
http.interceptors.response.use(
  (response) => {
    const res = response.data as BackendResponse
    if (res.code !== '00000') {
      console.error('API Error:', res.message)
      return Promise.reject(new Error(res.message || '请求失败'))
    }
    return res  // 返回整个响应对象，包含 data
  },
  async (error) => {
    if (error.response?.status === 401) {
      // Token 过期，尝试刷新
      const refreshToken = localStorage.getItem('refreshToken')
      if (refreshToken) {
        try {
          const res = await axios.post('/api/rag/v1/auth/refresh', {
            refreshToken
          })
          if (res.data.code === '00000') {
            localStorage.setItem('token', res.data.data.accessToken)
            // 重试原请求
            error.config.headers.Authorization = `Bearer ${res.data.data.accessToken}`
            return http.request(error.config)
          }
        } catch {
          // 刷新失败，跳转登录
          localStorage.clear()
          window.location.href = '/login'
        }
      }
    }
    return Promise.reject(error)
  }
)

export default http
```

### 5.2 SSE 流式解析

```typescript
// frontend/src/api/sse.ts
export interface SSECallbacks {
  onStart?: (data: { messageId: string; sessionId: string }) => void
  onDelta?: (data: { delta: string }) => void
  onCitation?: (data: { citations: Citation[] }) => void
  onDone?: (data: { usage?: any; finishReason?: string }) => void
  onError?: (error: { code: string; message: string }) => void
}

export function createChatStream(
  sessionId: string,
  query: string,
  kbId: string | undefined,
  callbacks: SSECallbacks
): () => void {
  const params = new URLSearchParams({ query })
  if (kbId) params.append('kbId', kbId)
  
  const token = localStorage.getItem('token')
  const url = `/api/rag/v1/sessions/${sessionId}/chat/stream?${params.toString()}`
  
  const eventSource = new EventSource(url, {
    headers: {
      Authorization: `Bearer ${token}`
    }
  })

  eventSource.addEventListener('start', (e) => {
    try {
      const data = JSON.parse(e.data)
      callbacks.onStart?.(data)
    } catch (error) {
      console.error('Failed to parse start event:', error)
    }
  })

  eventSource.addEventListener('delta', (e) => {
    try {
      const data = JSON.parse(e.data)
      callbacks.onDelta?.(data)
    } catch (error) {
      console.error('Failed to parse delta event:', error)
    }
  })

  eventSource.addEventListener('citation', (e) => {
    try {
      const data = JSON.parse(e.data)
      callbacks.onCitation?.(data)
    } catch (error) {
      console.error('Failed to parse citation event:', error)
    }
  })

  eventSource.addEventListener('done', (e) => {
    try {
      const data = JSON.parse(e.data)
      callbacks.onDone?.(data)
    } catch (error) {
      console.error('Failed to parse done event:', error)
    } finally {
      eventSource.close()
    }
  })

  eventSource.addEventListener('error', (e) => {
    try {
      const data = JSON.parse(e.data)
      callbacks.onError?.(data)
    } catch (error) {
      console.error('SSE connection error:', error)
      callbacks.onError?.({ code: 'SSE_ERROR', message: 'SSE 连接错误' })
    } finally {
      eventSource.close()
    }
  })

  // 返回取消函数
  return () => {
    eventSource.close()
  }
}
```

```typescript
async uploadDocument(
  kbId: string,
  file: File,
  options?: {
    title?: string
    tags?: string
    language?: string
    onProgress?: (progress: number) => void
  }
): Promise<BackendResponse<UploadDocumentResponseData>> {

  if (options?.title) formData.append('title', options.title)
  if (options?.tags) formData.append('tags', options.tags)
  if (options?.language) formData.append('language', options.language)

  return http.post(`/knowledge-bases/${kbId}/documents`, formData, {

    },
    onUploadProgress: (progressEvent) => {
      if (options?.onProgress && progressEvent.total) {
        const percent = Math.round((progressEvent.loaded * 100) / progressEvent.total)
        options.onProgress(percent)
      }
    }
  })
```

## 8. 常见陷阱与注意事项

1. **响应码类型**
   ```typescript
   // ❌ 错误
   if (response.code === 200) { }
   
   // ✅ 正确
   if (response.code === '00000') { }
   ```

2. **分页数据访问**
   ```typescript
   // ❌ 错误
   const items = response.data
   
   // ✅ 正确
   const items = response.data.items
   ```

3. **字段名混淆**
   ```typescript
   // ❌ 错误
   console.log(session.id)  // 后端返回的是 sessionId
   
   // ✅ 正确
   console.log(session.sessionId)
   // 或在 API 层映射后使用 session.id
   ```

4. **时间戳字段**
   ```typescript
   // ❌ 错误：直接使用可能为空的字段
   createdAt: item.gmtModified
   
   // ✅ 正确：提供默认值
   createdAt: item.gmtModified || item.gmtCreate || new Date().toISOString()
   ```

5. **SSE 事件监听**
   ```typescript
   // ❌ 错误：使用 onmessage
   eventSource.onmessage = (e) => { }
   
   // ✅ 正确：监听具体事件类型
   eventSource.addEventListener('delta', (e) => { })
   ```

6. **文件上传**
   ```typescript
   // ❌ 错误：忘记设置 Content-Type
   http.post('/upload', formData)
   
   // ✅ 正确
   http.post('/upload', formData, {
     headers: { 'Content-Type': 'multipart/form-data' }
   })
   ```

7. **Token 刷新**
   - 保存 refreshToken
   - 在 401 错误时自动刷新
   - 刷新失败时清除登录状态

8. **空值处理**
   ```typescript
   // ❌ 错误：未处理 undefined
   const name = doc.fileName
   
   // ✅ 正确：提供默认值
   const name = doc.fileName || doc.title || 'Unknown'
   ```

1. **理解差异是前提**
   - 响应格式差异（成功码、分页格式）
   - 字段名称差异（ID、时间戳、状态等）

   - 在 API 层统一处理响应
   - 在 API 模块层进行字段映射
   - 保持视图层代码简洁

3. **类型安全是保障**
   - 定义完整的 TypeScript 接口
   - 利用编译器发现问题
   - 避免运行时错误

4. **错误处理是关键**
   - 统一的错误拦截
   - 友好的错误提示
   - Token 自动刷新
