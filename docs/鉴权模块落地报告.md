# Auth 鉴权模块代码实现与验证报告

## 1. 已完成的开发项

按照早前经您授权的设计方案 `implementation_plan.md`，我们顺利完成了 Auth 模块的核心开发与架构调整，具体产出包含：

### 公共服务层 (`rag-common`)
- **常量定义**：新建 `AuthConstants`，统一定义了跨微服务所需的 JWT 相关常量与验证码 Redis 键结构。
- **JWT 生成器**：新建 `JwtUtils`，统一提供 HS256 签名的 JWT 生成、解析及内容提取功能。
- **业务异常**：添加 `AuthException`，用于鉴权失败时的业务中断反馈。
- **依赖优化**：删除了 `lombok` 的 `optional` 制约，让 `rag-auth` 与后续其他微服务可以直接继承公共包里的简化注解工具。

### 鉴权业务模块 (`rag-auth`)
- **实体流转配置**：引入 `hutool-captcha` 生成轻量级图形验证码；通过 `User` 实体模型和 `UserMapper` 将数据库数据接驳到验证逻辑。
- **验证码业务**：在 `AuthServiceImpl` 中实现了防刷限流机制，同 IP 每分钟最多获取 10 次，图片验证码暂存于 Redis 5分钟。
- **登录核验业务**：接收账号密码和验证码，利用 `StringRedisTemplate` 进行【阅后即焚】式的验证码一次性比对，随后通过 `BCryptPasswordEncoder` 安全比对密码哈希值并签发 Token。

### 网关代理模块 (`rag-gateway`)
- **统一防护**：开发 `AuthGlobalFilter` 实现最高优先级的网关控制拦截器。在排除 `WhiteListConfig` 注入的白名单路径后，对剩余所有请求剥离 Header 并提取解析 Token。
- **透传机制**：Token 验真通过后拆解出 `UserId` 并放进 `X-User-Id` 请求头部向下游微服务透传，实现了“网关一次鉴权，微服务全程信任”的企业级规范。

## 2. 验证结果

- 通过了 `mvn clean compile` 的所有子模块级联自动编译与接口对接验证（包含 Lombok 代码生成器的自动处理环节）。
- 代码语法与 Spring Boot / Spring Cloud 机制的依赖关联完整。
- **暂不可运行时测试**：因本地尚未提供运行时所需的 PostgreSQL、Nacos 或 Redis 实例连接参数，目前的验证仅涵盖静态依赖树与语义级检测。
