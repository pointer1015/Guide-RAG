# Auth 鉴权模块实现方案

本文档描述了微服务架构下工业级多模态 RAG 系统的 Auth 鉴权模块代码实现方案。基于 **Spring Security 思想 + JWT + 网关统一鉴权**，包含邮箱登录、BCrypt 密码加密与 Redis 结合的图形验证码限频防刷机制。

## Proposed Changes

### 1. 业务与技术链路设计

符合《开发文档.md》中定义的微服务架构规范：
1. **身份认证 (`rag-auth`)**：提供获取图形验证码、邮箱登录功能。通过查库比对 BCrypt 密码，校验成功后签发 JWT (Access Token)。
2. **统一鉴权 (`rag-gateway`)**：作为所有请求的第一道防线。除了 `/auth/**` 等白名单路径外，全部进行 JWT 验签。解析有效后将 `userId` 等信息放入 HTTP Header 传递给下游微服务。

### 2. 核心功能设计细则

#### A. 图形验证码与防刷 (Captcha)
- **工具选型**: 使用 `Hutool-captcha` 生成 4 位字母+数字混合图片验证码。
- **流程**:
  1. 客户端请求 `/auth/captcha`。
  2. 获取请求的真实 IP 地址。基于 Redis + Lua 脚本进行接口限流（如：同 IP 每分钟最多获取 10 次），防恶意刷流。
  3. 服务端生成验证码组合与唯一 `uuid`，将 `code` 存入 Redis，Key 为 `auth:captcha:{uuid}`，过期时间设置 5 分钟。
  4. 响应 `Base64图片格式` 和 `uuid` 给客户端。

#### B. 用户登录与密码加密 (Login)
- **接口设计**: `POST /auth/login`
- **入参**: `email` (邮箱)，`password` (原文密码)，`captchaCode` (验证码)，`captchaUuid` (验证码业务流水号)。
- **流程**:
  1. **验证码校验**: 先通过 Redis 读取对应 `uuid` 的验证码进行全转小写比对，**比对后立即从 Redis 移除**（阅后即焚，防止重复提交）。
  2. **密码校验**: 根据 `email` 查库提取 `user` 实体。使用 `BCryptPasswordEncoder.matches()` 将前端传来的明文与数据库中加密后的 `password_hash` 进行比对。
  3. **下发 Token**: 密码校验通过后，使用 JWT 签发 `accessToken` (通常 2 小时过期) 和 `refreshToken` (通常 7 天过期)。

### 3. 文件修改计划 (待执行)

#### rag-auth (认证服务模块)
实现登录、验证签发的核心逻辑：
- `pom.xml`: 引入 `jjwt`, `hutool-captcha`, `spring-boot-starter-data-redis`。
- `com.guide.rag.auth.controller.AuthController`: 暴露 `/captcha` 和 `/login` 接口。
- `com.guide.rag.auth.service.AuthService`: 登录验证与验证码获取服务。
- `com.guide.rag.auth.util.JwtUtils`: JWT 生成工具类。

#### rag-gateway (网关鉴权模块)
实现基于全局过滤器的无状态身份鉴权拦截：
- `pom.xml`: 引入 `jjwt`, `spring-boot-starter-data-redis`。
- `com.guide.rag.gateway.filter.AuthGlobalFilter`: 全局异常拦截，解析 HTTP Header 中 `Authorization: Bearer <token>` 字段。
- 校验 Token 合法性，并从 Redis 读取黑名单（支持后续登出功能）。
-校验成功后将解析出的 `userId` 推入 `ServerHttpRequest` Header 透传给下游。
- `com.guide.rag.gateway.config.WhiteListConfig`: 配置不需要拦截的白名单路由（如 `/auth/**`）。

#### rag-common (公共模块)
提取通用依赖及异常常量设计，复用到各个微服务中：
- `com.guide.rag.common.constant.AuthConstants`: 定义 Token Header 键名、Redis 缓存 Key 格式等企业级通用常量。
- `com.guide.rag.common.exception.AuthException`: 鉴权等异常类统一抛出和捕获。

## 验证与验收目标 
1. **防刷验证**: 验证同一个 IP 一分钟请求图形验证码超过 10 次报错。
2. **流程验证**: 只有输入正确的图形验证码及正确邮箱密码组合才能换取 Token。
3. **网关拦截测试**: 携带伪造、过期或者无 Token 请求其他微服务被网关直接拒绝 (HTTP 401)。携带有效 Token 会经过网关透传头信息到下游。
